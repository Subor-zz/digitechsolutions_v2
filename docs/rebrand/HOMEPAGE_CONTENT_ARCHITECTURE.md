# Homepage Content Architecture

## Editorial rule

The homepage is a concise decision path, not a compressed version of every service page. Target nine chapters with short copy, one dominant visual per chapter and deep links for details. On mobile, all meaning exists in normal document flow before enhancement.

## Chapters

### 1. Entry

- **Purpose / insight:** Immediate brand and readiness; visitors should never wait for a reveal.
- **Draft:** Small descriptor `Workflow- en applicatiemodernisering`; no separate marketing headline.
- **Visual / interaction:** Fragmented Flow Line resolves to a stable start node in ≤1 s; skip if already visited is optional.
- **Mobile / reduced motion:** Final line state visible immediately; no timed interstitial.
- **Proof / CTA / dependency:** None; header scan CTA is usable throughout.

### 2. Conflict

- **Purpose / insight:** Name the shared operational problem and category within five seconds.
- **Headline:** `Moderniseer wat je organisatie vertraagt.`
- **Support:** `Digitech Solutions brengt vastgelopen workflows en verouderde bedrijfsapplicaties terug naar een beheersbare route—van analyse en scope tot gerichte uitvoering en overdracht.`
- **Visual / interaction:** A simple system map shows inbox, spreadsheet, handoff and aging application feeding one constrained flow. Line progress supports, never gates, reading.
- **Mobile / reduced motion:** Static stacked map with two labelled problem routes; no pinned hero.
- **Proof / CTA / dependency:** Claim needs comprehension testing. CTA `Start met een Modernization Scan`; secondary `Bekijk de werkwijze`. Depends on approved descriptor.

### 3. Hidden Drag

- **Purpose / insight:** Show that cost and risk accumulate between tools and people.
- **Headline:** `De grootste vertraging zit vaak tussen je systemen.`
- **Support:** `In overdrachten, workarounds, dubbele invoer en uitzonderingen die pas laat zichtbaar worden.`
- **Visual / interaction:** Before/after workflow; a short scrub highlights wait, repeat entry, missing trail and one-person dependency, then a controlled flow with a human check.
- **Mobile / reduced motion:** Two static diagrams plus a four-item friction list; optional tap steps, never horizontal-only.
- **Proof / CTA / dependency:** Generic illustrative diagram labelled “voorbeeld”, not a case. CTA `Herken je dit? Bekijk workflowmodernisering`.

### 4. Diagnosis

- **Purpose / insight:** Reassure both buyers that Digitech does not prescribe automation or a rewrite.
- **Headline:** `Eerst begrijpen. Dan pas automatiseren of herbouwen.`
- **Support:** `De scan bekijkt proces, data, systemen, risico en eigenaarschap voordat een route wordt gekozen.`
- **Visual / interaction:** Five-layer stack and decision lens: simplify, integrate, automate, secure, refactor, replatform, rebuild, retire. Selecting a layer reveals one sentence; all sentences remain in accessible DOM.
- **Mobile / reduced motion:** Ordered accordion/list, default first item open; no spatial dependency.
- **Proof / CTA / dependency:** Requires example scan excerpt before visual design. CTA `Bekijk de Modernization Scan`.

### 5. Two Routes

- **Purpose / insight:** Allow self-selection without splitting the brand story.
- **Headline:** `Eén moderniseringsvraag. Twee expertise-routes.`
- **Support:** Workflow: `Van handwerk naar een beheersbare workflow.` Application: `Van kwetsbare legacy naar software die weer vooruit kan.`
- **Visual / interaction:** Two equal routes share one incoming Flow Line; focus/tap reveals fit signals and one output. “Niet zeker” keeps the line combined.
- **Mobile / reduced motion:** Stacked cards with explicit headings and a third “workflow en applicatie zitten samen vast” link.
- **Proof / CTA / dependency:** CTAs to both service pages; needs fit/non-fit copy, not cases.

### 6. Delivery Method

- **Purpose / insight:** Convert speed skepticism into a controlled method.
- **Headline:** `Snel waar het kan. Gecontroleerd waar het moet.`
- **Support:** `Scope → Map → Simplify → Build → Verify → Transfer.` One plain-language sentence per step.
- **Visual / interaction:** Flow Line passes through gates; only the active gate gains emphasis. No autoplay loop.
- **Mobile / reduced motion:** Numbered vertical sequence with artifacts and trust gates visible.
- **Proof / CTA / dependency:** Proof requires sample artifacts/process evidence. CTA `Bekijk de werkwijze`.

### 7. Proof

- **Purpose / insight:** Let claims be inspected, including exclusions.
- **Headline:** `Geen toekomstverhaal zonder bewijs.`
- **Support:** Case card shows classification, starting state, observed evidence, decision, delivered scope, result and what was deliberately not built.
- **Visual / interaction:** One featured workflow and one application case; optional inline expansion only if it improves comparison.
- **Mobile / reduced motion:** Normal cards/links, no carousel requirement.
- **Proof / CTA / dependency:** Blocked until publishable cases exist. CTA `Bekijk de case(s)`. If absent at launch design time, use an explicitly labelled report anatomy—not a fictional result.

### 8. Founder-led Trust

- **Purpose / insight:** Resolve accountability and solo-capacity concerns.
- **Headline:** `Geen overdracht naar een anoniem deliveryteam.`
- **Support:** `Je werkt rechtstreeks met Subor Cheung: van analyse en requirements tot uitvoering en overdracht. Specialistische expertise wordt alleen transparant toegevoegd waar die nodig is.`
- **Visual / interaction:** Real portrait/work-context photo with capability and boundary annotations; restrained parallax is optional and nonessential.
- **Mobile / reduced motion:** Static responsive crop and text.
- **Proof / CTA / dependency:** Verified bio, photo, credentials and partner language. CTA `Over Subor`.

### 9. Modernization Scan CTA

- **Purpose / insight:** Make the safe first step tangible.
- **Headline:** `Je hoeft nog niet te weten wat er gebouwd moet worden.`
- **Support:** `De Modernization Scan brengt frictie, risico’s en afhankelijkheden in kaart en vertaalt ze naar een gefaseerde route die je ook zonder vervolgopdracht kunt gebruiken.`
- **Visual / interaction:** Flow Line ends as a three-part roadmap: current state → decisions → first step. Route choice pre-fills intake only with user action.
- **Mobile / reduced motion:** Static deliverables list and large CTA.
- **Proof / CTA / dependency:** Needs confirmed inputs, duration and commercial presentation before final copy. CTA `Start met een Modernization Scan`; fallback `Plan een verkennend gesprek`.

## Move to service/detail pages

Keep detailed deliverables, stack names, code/security checklists, migration/rollback, integration patterns, AI methods, all FAQs, scan commercial terms and full case narratives off the homepage. Homepage proof should show one decisive fact per case, never a dense gallery.

## Low-fidelity acceptance

The page is understandable with CSS/JavaScript disabled; primary CTA is available above the fold and at the end; operations understands the opening while IT sees technical control by Diagnosis; no chapter depends on a case that does not exist; mobile preserves sequence; reduced-motion is a designed state; and 5-second/30-second comprehension is tested.
